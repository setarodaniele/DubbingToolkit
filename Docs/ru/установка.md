# Установка

---

## Системные требования

| Компонент | Требование |
|-----------|------------|
| Операционная система | Windows (10 или 11) |
| PowerShell | Версия 5.1 или выше |
| Python | Python 3.11 — включён в папку `Installation/`, устанавливать отдельно не нужно |
| ffmpeg | Версия 7.1.1 — включена в `Tools/ffmpeg-7.1.1-full_build/`, настраивается автоматически |
| Интернет | Требуется для вызовов TTS API и первичной загрузки моделей перевода |

> Python берётся из `Installation/Python311/`, поэтому устанавливать Python в систему **не нужно**.

## Учётные данные TTS

Для использования синтеза речи необходимо предоставить учётные данные хотя бы одного провайдера.

### Структура папки `credentials/`

```
credentials/
├── azure_speech_credentials.json           ← ваши данные Azure
├── azure_speech_credentials.template.json  ← шаблон (не изменять)
├── google_speech_credentials.json          ← ваши данные Google
└── google_speech_credentials.template.json ← шаблон (не изменять)
```

### Быстрая настройка

1. Найдите нужный файл шаблона в папке `credentials/`
2. Скопируйте его, убрав из имени `.template`
3. Откройте скопированный файл и заполните поля учётными данными

Подробные инструкции:
- Azure: [credenziali_azure.md](credenziali_azure.md)
- Google: [credenziali_google.md](credenziali_google.md)

> Функции извлечения аудио, транскрипции и перевода работают **без** учётных данных TTS.

## Виртуальное окружение Python

Управление виртуальным окружением полностью автоматизировано.

- Launcher создаёт и активирует `venv/` при первом запуске
- Все зависимости перечислены в `Scripts/requirements.txt`
- PyTorch устанавливается через специальный индекс пакетов, настроенный в `InstallDependencies.ps1`

### Сброс окружения

Если окружение повреждено или возникли проблемы с зависимостями:

```powershell
# Автоматический сброс
Scripts/reset_env.ps1
```

Или вручную:
```
1. Удалите папку venv/
2. Запустите StartDubbing.bat — окружение будет создано заново
```

## Настройки

Настройки проекта хранятся в папке `Settings/`:

| Файл | Назначение |
|------|------------|
| `Settings/settings.json` | Активная конфигурация (редактируется системой) |
| `Settings/settings_default.json` | Эталонные настройки по умолчанию — **не изменять** |
| `Settings/reset.json` | Параметры для сброса настроек |

### Основные параметры `settings.json`

```json
{
  "interface_lang": "en",
  "model": "small",
  "Transcript_Audio_Spoken_Lang": "it",
  "Translation_Target_Lang": "en",
  "Dubbing_Lang": "en"
}
```

| Параметр | Описание |
|----------|----------|
| `interface_lang` | Язык интерфейса (`en`, `it`, `ru`, и др.) |
| `model` | Модель Whisper для транскрипции (`tiny`, `base`, `small`, `medium`, `large`) |
| `Transcript_Audio_Spoken_Lang` | Язык исходного аудио |
| `Translation_Target_Lang` | Целевой язык перевода |
| `Dubbing_Lang` | Язык озвучивания |

> **Важно:** настройки не сохраняются между сессиями — все предпочтения нужно выбирать заново при каждом запуске. Постоянное сохранение запланировано на будущее.

## Запуск

```bat
StartDubbing.bat
```

При запуске происходит следующее:

1. Активируется встроенный Python runtime из `Installation/Python311/`
2. Создаётся или активируется виртуальное окружение `venv/`
3. Проверяются учётные данные (проблемы отображаются в меню перед TTS)
4. Запускается главный интерфейс (`Scripts/Regista.py`)

> Если учётные данные TTS отсутствуют или недействительны, об этом будет сообщено в меню. При этом функции извлечения аудио, транскрипции и перевода **остаются доступными**.

## Перемещение и удаление

**Перемещение проекта** не рекомендуется, но возможно:

1. Удалите папку `venv/`
2. Запустите `StartDubbing.bat` в новом расположении — окружение будет создано заново

**Удаление проекта:**

Просто удалите всю папку `DubbingToolkit/`. Никаких файлов за пределами этой папки проект не создаёт.
