# Настройка учётных данных Google

Это руководство описывает, как получить и настроить учётные данные для Google Cloud Text-to-Speech, необходимые для синтеза речи через провайдера Google.

> В будущем будет доступно подробное видеоруководство по созданию учётных данных Google Cloud.

---

## Файл учётных данных

Файл для заполнения:

```
credentials/google_speech_credentials.json
```

Требуемая структура:

```json
{
  "type": "service_account",
  "project_id": "YOUR_PROJECT_ID",
  "private_key_id": "YOUR_PRIVATE_KEY_ID",
  "private_key": "YOUR_PRIVATE_KEY",
  "client_email": "YOUR_CLIENT_EMAIL",
  "client_id": "YOUR_CLIENT_ID",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
  "client_x509_cert_url": "YOUR_CLIENT_CERT_URL",
  "universe_domain": "googleapis.com"
}
```

Файл шаблона с этой структурой уже доступен в проекте:

```
credentials/google_speech_credentials.template.json
```

Скопируйте его, переименуйте, удалив расширение `.template`, и заполните поля значениями вашей сервисной учётной записи.

---

## Обязательные поля

Поля с фиксированным значением (`type`, `auth_uri`, `token_uri`, `auth_provider_x509_cert_url`, `universe_domain`) не нужно изменять — они одинаковы для всех аккаунтов Google Cloud.

Поля для заполнения:

| Поле | Описание |
|---|---|
| `project_id` | ID проекта Google Cloud |
| `private_key_id` | ID закрытого ключа сервисной учётной записи |
| `private_key` | Закрытый ключ в формате PEM (включая заголовки `-----BEGIN/END PRIVATE KEY-----`) |
| `client_email` | Электронная почта сервисной учётной записи (например, `имя@проект.iam.gserviceaccount.com`) |
| `client_id` | Числовой ID сервисной учётной записи |
| `client_x509_cert_url` | URL сертификата X509 сервисной учётной записи |

## Как получить учётные данные

1. Войти в [Google Cloud Console](https://console.cloud.google.com)
2. Создать или выбрать существующий проект
3. Включить API **Cloud Text-to-Speech** в разделе **API и сервисы**
4. Перейти в **IAM и администрирование → Сервисные учётные записи**
5. Создать новую сервисную учётную запись с ролью **Cloud Text-to-Speech Agent** (или эквивалентной с доступом TTS)
6. На вкладке **Ключи** сервисной учётной записи создать новый ключ в формате **JSON**
7. Скачать сгенерированный JSON-файл — он содержит все необходимые поля уже заполненными
8. Скопировать содержимое скачанного файла в `credentials/google_speech_credentials.json`

---

## Примечания

- JSON-файл, скачанный из Google Cloud Console, уже находится в правильном формате и может использоваться напрямую без изменений.
- Храните файл в безопасном месте: закрытый ключ не может быть восстановлен после создания.
- В случае недействительных или истёкших учётных данных создайте новый ключ в Console и обновите файл.
- Сервис Cloud Text-to-Speech предоставляет бесплатный тариф с ежемесячным лимитом символов.
