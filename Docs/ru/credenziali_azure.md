# Учётные данные Azure

---

## Предупреждение безопасности

> **Учётные данные являются персональными и привязаны к вашему биллингу.**
>
> - Никогда не публикуйте их в общедоступных репозиториях
> - Никогда не отправляйте их в чатах или по электронной почте
> - При подозрении на компрометацию немедленно отзовите ключи в Azure Portal

---

## Файл учётных данных

**Путь:** `credentials/azure_speech_credentials.json`

**Структура:**

```json
{
  "subscription": "YOUR_AZURE_SPEECH_KEY",
  "region": "YOUR_AZURE_REGION"
}
```

---

## Настройка по шаблону

В папке `credentials/` уже есть готовый шаблон:

```
credentials/azure_speech_credentials.template.json
```

Порядок настройки:

1. Скопируйте шаблон: `azure_speech_credentials.template.json`
2. Переименуйте копию, убрав `.template`: `azure_speech_credentials.json`
3. Откройте файл и заполните поля `subscription` и `region`

---

## Описание полей

### `subscription`

Ключ API из Azure Portal.

- Находится в разделе **«Ключи и конечная точка»** (Keys and endpoint) вашего ресурса Speech
- Представляет собой строку из 32 шестнадцатеричных символов
- Azure предоставляет два ключа (`KEY 1` и `KEY 2`) — можно использовать любой

### `region`

Регион Azure, в котором создан ресурс Speech.

- Должен **точно совпадать** с регионом ресурса
- Примеры: `westeurope`, `eastus`, `northeurope`, `southeastasia`
- Регион указан на странице ресурса в Azure Portal

---

## Как получить учётные данные

1. Откройте [Azure Portal](https://portal.azure.com)
2. Создайте новый ресурс: **Cognitive Services → Speech**
3. После создания ресурса откройте его страницу
4. Перейдите в раздел **«Ключи и конечная точка»** (Keys and endpoint)
5. Скопируйте **KEY 1** или **KEY 2**
6. Запишите **регион** (отображается рядом с ключами)
7. Вставьте ключ и регион в файл `credentials/azure_speech_credentials.json`

## Дополнительные сведения

- Ключ подписки Azure — это ровно 32 шестнадцатеричных символа
- Azure Speech предоставляет **бесплатный тарифный план** с ежемесячным лимитом символов
- Если учётные данные недействительны, система сообщит об этом в меню перед запуском TTS
- Функции извлечения аудио, транскрипции и перевода работают без учётных данных Azure
