# 安装与配置

本指南介绍 DubbingToolkit 的系统要求、凭据结构和初始设置流程。

---

## 系统要求

- **操作系统：** Windows 与 PowerShell 5.1
- **Python：** 包含在项目中，无需系统级安装
  - 当前使用 Python 3.11，位于 `Installation/` 文件夹中，由系统内部管理。
- **ffmpeg：** 包含在 `Tools/ffmpeg-7.1.1-full_build/` 中，无需单独安装
- **网络连接：** 访问外部资源的模块需要网络，包括 TTS API（Azure、Google）以及首次运行时的翻译模型下载。

---

## TTS 凭据

TTS 提供商的凭据必须放置在 `credentials/` 文件夹中。目前支持 Azure 和 Google 两种提供商，各自使用独立的 JSON 文件。

| 文件 | 提供商 |
|---|---|
| `azure_speech_credentials.json` | Azure Cognitive Services Speech |
| `google_speech_credentials.json` | Google Cloud TTS |

每种提供商均提供包含所需结构的模板文件：

```
credentials/azure_speech_credentials.template.json
credentials/google_speech_credentials.template.json
```

复制模板文件，去掉 `.template` 扩展名，然后在生成的文件中填写您的凭据。

---

## 虚拟环境与 Python 依赖

项目启动时，启动器会自动处理虚拟环境的创建和激活，以及依赖项的安装。正常情况下无需手动干预。

主要依赖项列在 `Scripts/requirements.txt` 中。

### 重置虚拟环境

**从项目内部重置**（项目必须能够启动）：

从界面选择重置选项，或直接运行：
```powershell
Scripts/reset_env.ps1
```
该脚本会自动重建 venv 并重新安装依赖项。

**完全手动重置**（项目无法启动时）：

手动删除 `venv/` 文件夹。下次通过 `StartDubbing.bat` 启动时，启动器会检测到 venv 缺失并自动重建。

## 初始配置

项目的运行设置位于 `Settings/` 目录中：

| 文件 | 用途 |
|---|---|
| `settings.json` | 活动配置 |
| `settings_default.json` | 参考配置（请勿修改） |
| `reset.json` | 重置参数 |

### `settings.json` 中的主要参数

- **`interface_lang`** — 界面语言（例如 `"it"`、`"en"`、`"zh"`）
- TTS 提供商设置（活动提供商、选定语音、目标语言）
- Whisper 转录参数（模型、语言）

> **注意：** 目前设置在会话之间不持久保存。每次启动时会提示选择界面语言，所有其他设置将重置为默认值。设置持久化计划作为未来改进实现。

---

## 启动与自动初始化

用户通过以下方式启动项目：

```
StartDubbing.bat
```

启动器随后会自动执行以下步骤，无需用户干预：

1. 验证并激活本地 Python 运行时
2. 创建或激活虚拟环境
3. API 凭据检查
4. 启动主界面

如果凭据缺失或无效，系统会在授予 TTS 功能访问权限之前在菜单中报告此情况。
其他功能——音频提取、转录和翻译——不依赖凭据，保持可访问状态。

---

## 移动项目和卸载

虽然不推荐，但项目可以移动到磁盘上的不同位置或另一台机器，但每次移动后必须重建虚拟环境，因为 venv 包含与原始位置绑定的绝对路径。操作步骤：

1. 删除 `venv/` 文件夹
2. 启动 `StartDubbing.bat` — 启动器将重建 venv 并重新安装依赖项

如果项目通过发行包安装，移动后 Windows **应用和功能**中的条目将不再有效。您无需使用它：要卸载项目，只需删除整个文件夹即可。项目不会在文件夹外部写入任何文件。
